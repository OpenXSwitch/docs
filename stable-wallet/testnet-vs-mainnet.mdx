<Note>
  Within the **Smart Accounts** section of the OpenXSwitch platform, you can **switch** between **mainnet and testnet** environments to separate production activity from testing workflows.
</Note>

### Mainnets

Mainnets are used for production-grade transactions and real-world interactions through OpenXSwitch APIs, including:

- Sending real crypto funds (e.g., withdrawals, payments)
- Reading onchain data for dashboards, reporting, and analytics

OpenXSwitch APIs also streamline production workflows by enabling direct access to balances, transactions, and event data without requiring users to run or maintain blockchain nodes.

### Testnets

Testnets are used for development, testing, and QA in a risk-free environment where no real funds are involved:

- Testing smart contract deployments before mainnet release
- Simulating end-to-end workflows using testnet tokens
- Running demos and sandbox environments for validation purposes

### Testnet Faucets

To test payment flows, you will need testnet tokens:

- [Circle Faucet](https://faucet.circle.com) - testnet USDC and EURC for supported testnet blockchains.
- [Google Web3 Faucet](https://cloud.google.com/application/web3/faucet) - testnet ETH, CELO, PYUSD.
- Native gas tokens can be obtained from blockchain-specific faucets listed in each testnet’s official documentation.
